GitHub Basics

GitHub is a platform for hosting and collaborating on code repositories using Git. It allows developers to manage version control, collaborate with others, and deploy code seamlessly.

Steps to Initialize, Add, Commit, Push Code, and Making and Merging Branches

  1. Initialize a Git repository:
    Use git init to initialize a new Git repository in your project folder.
    git init
  2. Make a new branch:
    Use git checkout-b (branch name) to create a new branch.
    git checkout-b (branch name)
  3. Add files to staging:
    Use git add to stage files for a commit. To stage all files:
    git add .
  4. Commit your changes:
    Use git commit to save your changes with a message describing the changes.
    git commit -m "Your commit message here"
  5. Switch Branches:
    Use git checkout (branch name) to switch branches.
    git checkout (branch name)
  6. Merge branches:
    Use git merge (branch name) to merge the branches.
    git merge (branch name)
  7. Push changes to a remote repository:
    Add a remote repository (if not already added) using:
    git remote add origin https://github.com/username/repository.git
    Push your changes to the remote repository using:
    git push -u origin main

Example Workflow

git init
git checkout-b (branch name)
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/my-project.git
git checkout (branch name)
git merge (branch name)
git push -u origin main
